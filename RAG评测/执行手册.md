# 执行手册

从零到一跑完一套完整评测的命令清单。约 **1.5 小时、$5-7**。

## 前置

- Python 3.13 venv 在 `backend/venv/`（项目默认）
- `lightrag-hku / httpx / ragas` 已装（见 `backend/requirements.txt`）
- `backend/.env` 里配 `DEEPSEEK_API_KEY` + `MOONSHOT_API_KEY`
- MongoDB + Qdrant 数据目录已初始化（跑过一次 `start-all.sh`）

## Step 1 — 准备数据（~30s，一次性）

```bash
cd backend
venv/bin/python scripts/prep_crud_eval.py
```

第一次会下载 `split_merged.json`（~26MB，走 GitHub raw，不需翻墙）。

产出：
- `tests/eval-data/crud-mini/docs/crud-NNNN.txt`（118 篇）
- `tests/eval-data/crud-mini/queries.jsonl`（60 条）
- `tests/eval-data/crud-mini/metadata.json`

**验证**：`jq '.total_queries, .total_unique_docs' tests/eval-data/crud-mini/metadata.json` 应打印 `60 / 118`。

## Step 2 — 启动服务

```bash
# 项目根
cd /Users/admin/Desktop/personal/langchain-rag-knowledge-base-chatbox
bash start-all.sh
```

等到：
- <http://localhost:8000/docs> 可访问
- 前端 <http://localhost:5173/> 可打开

## Step 3 — 激活 Embedding（BGE-M3）

```bash
curl -X POST http://localhost:8000/api/embedding/activate
# 或前端点 /knowledge 页面的"启用 Embedding"按钮
```

**验证**：
```bash
curl -s http://localhost:8000/api/embedding/system-info | \
  python3 -c "import sys,json; print(json.load(sys.stdin)['embedding_ready'])"
# 应输出 True
```

## Step 4 — 批量入库 + 构建 Graph RAG 索引

```bash
cd backend
venv/bin/python scripts/ingest_eval_kb.py --build-graph --yes --concurrency 2
```

- **上传 118 篇**：~3-5 min（受 Embedding 推理速度限制，MPS 约 20-40 chunks/s）
- **构建 Graph RAG**：~5-15 min，<$1（LightRAG 对每篇调 1 次 DeepSeek-chat 抽实体关系）

`--yes` 跳过 Graph RAG 的成本二次确认。如果只想做 non-graph 评测，省略 `--build-graph`。

**验证**：
```bash
# 所有文档都 done
curl -s http://localhost:8000/api/documents | \
  python3 -c "import sys,json; d=json.load(sys.stdin); print(sum(1 for x in d if x['status']=='done'), '/', len(d))"

# Graph RAG 统计
curl -s http://localhost:8000/api/embedding/graph-rag/stats
# 期望：{"exists":true, "nodes":>0, "edges":>0, "documents":118, ...}
```

## Step 5 — Smoke（先跑 5 条 × 2 配置）

评测链路多（SSE / 切档 / ragas），先用小样本验证：

```bash
venv/bin/python scripts/run_rag_bench.py \
  --max-queries 5 \
  --configs off graph_hybrid
```

产出：`tests/eval-data/results/run_<ts>.jsonl`（10 行）。

**验证**：`wc -l tests/eval-data/results/latest.jsonl` 应为 10；查一眼有没有 `"error":` 非 null。

## Step 6 — 全量跑（10 × 60 = 600 trials）

```bash
venv/bin/python scripts/run_rag_bench.py
```

~40-60 min，~$2-3。终端每秒打一行 `[N/600] ✓ config qid category Xms`，失败会打 `!`。

**断点续跑**：如果中途挂了，**本 runner 不支持断点续跑**（因为 MongoDB 每次新建 conversation，重跑会重复）。如需中断重来，先 truncate 对应 jsonl 或改脚本加 `--resume-from`。

## Step 7 — 出报告（加 RAGAS 分）

```bash
# 用最近一次 run（latest.jsonl 软链）
venv/bin/python scripts/score_rag_bench.py

# 或只看客观 Recall@K 跳过 RAGAS（秒出）
venv/bin/python scripts/score_rag_bench.py --skip-ragas
```

RAGAS 会跑 1200 次 judge（600 × 2 指标），~70 min。产出：
- `tests/eval-data/results/report_<ts>.md`——markdown 对比表
- `tests/eval-data/results/scored_<ts>.jsonl`——带分数的明细 JSONL

**查看**：
```bash
open tests/eval-data/results/report_*.md   # macOS 默认 markdown 浏览器
```

## 常见问题

### Q1：`embedding not ready` 报错

**原因**：忘了激活 Embedding 模型，或模型还在加载。
**解法**：Step 3 的 curl 先跑；等 `embedding_ready: true` 再继续。

### Q2：Graph RAG 配置跑出来 `refs=0, recall=None`

**原因**：LightRAG 的 `---Sources---` 解析失败（某些版本输出格式不是标准 markdown table）。
**解法**：
1. 打开 `backend/app/services/graph_rag.py::_split_by_sources`，加一条 print 看 LightRAG 实际返回格式
2. 或更稳：手工 diff `scored_<ts>.jsonl` 里某条 Graph RAG trial 和同 query 的 `off` trial，看 `answer` 内容是否有"实体关系"痕迹。RAGAS 的 faithfulness / correctness 仍然有效，只是 Recall@K 失去意义。

### Q3：RAGAS 跑到一半挂掉

**原因**：DeepSeek API 偶尔 timeout 或限流。
**解法**：`score_rag_bench` 已开 `raise_exceptions=False`，单条失败该指标标 None；重跑 `scripts/score_rag_bench.py --run results/run_<ts>.jsonl` 会重新算全部 RAGAS（不复用之前的分数，是 ragas 本身没有增量 API）。

### Q4：Neural engine / MPS OOM

**症状**：Embedding 激活后 ingest 到一半崩了。
**解法**：`ingest_eval_kb.py --concurrency 1`；或在 `/knowledge` 页关掉 Reranker（ingest 阶段本来就不需要）。

### Q5：想换一批 query 重测

```bash
# 新 seed
venv/bin/python scripts/prep_crud_eval.py --seed 2026

# KB 要重建（docs 变了）
curl -s http://localhost:8000/api/documents | \
  python3 -c "import sys,json; [print(d['file_id']) for d in json.load(sys.stdin)]" | \
  xargs -I {} curl -X DELETE http://localhost:8000/api/documents/{}
curl -X POST -H 'Content-Type: application/json' \
     -d '{"confirm":true}' \
     http://localhost:8000/api/embedding/graph-rag/clear

venv/bin/python scripts/ingest_eval_kb.py --build-graph --yes
```

## 一次性命令串联（复制即可跑）

```bash
# 假设项目根目录
cd /Users/admin/Desktop/personal/langchain-rag-knowledge-base-chatbox
bash start-all.sh &                                      # 后台起服务
until curl -sf http://localhost:8000/api/embedding/system-info >/dev/null; do sleep 2; done
curl -X POST http://localhost:8000/api/embedding/activate

cd backend
venv/bin/python scripts/prep_crud_eval.py
venv/bin/python scripts/ingest_eval_kb.py --build-graph --yes
venv/bin/python scripts/run_rag_bench.py --max-queries 5 --configs off graph_hybrid   # smoke
venv/bin/python scripts/run_rag_bench.py                                               # 全量
venv/bin/python scripts/score_rag_bench.py
```
