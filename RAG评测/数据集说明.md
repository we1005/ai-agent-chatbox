# 数据集 · CRUD-RAG mini

## 1. 选型

**CRUD-RAG**（IAAR-Shanghai 中文 RAG 评测基准，Apache 2.0）——
论文：[CRUD-RAG: A Comprehensive Chinese Benchmark for RAG](https://arxiv.org/abs/2401.17043)
代码仓：<https://github.com/IAAR-Shanghai/CRUD_RAG>

选它是因为：

- 纯中文、新闻语料，实体关系丰富（人物 / 机构 / 时间 / 事件）→ 能同时测试 vector RAG 与 Graph RAG
- 原生四类任务天然映射到本项目的策略档位：
  - `questanswer_1doc`   单文档 QA      → classical / Multi-Query 主场
  - `questanswer_2docs`  双文档 QA      → Agentic grading + rewrite 主场
  - `questanswer_3docs`  三文档 QA      → Graph RAG hybrid 主场
  - `hallu_modified`     续写/幻觉      → 未采用，下次再上
- 每条 QA 内嵌 `news1/news2/news3` 原文 → 无需拉 80k 的完整语料库，抽样即可得到 self-contained 的 KB
- 每条 QA 有 `questions` + `answers` ground truth → 可同时算 Recall@K（客观）+ RAGAS（LLM judge）

## 2. 原始数据源

```
https://raw.githubusercontent.com/IAAR-Shanghai/CRUD_RAG/main/data/crud_split/split_merged.json
```

- 大小：~26 MB
- 结构（顶层 6 个 key）：
  ```
  event_summary       : 2000  # 事件 + 摘要
  continuing_writing  : 1999  # 续写
  hallu_modified      : 1268  # 幻觉注入
  questanswer_1doc    :  800  # 本评测采样来源 ①
  questanswer_2docs   :  797  # 本评测采样来源 ②
  questanswer_3docs   :  797  # 本评测采样来源 ③
  ```
- 单条 QA 样例（`questanswer_1doc` 字段）：
  ```json
  {
    "ID": "64fa9b27b82641eb8ecbe14c",
    "event": "2023年7月28日，国家卫生健康委在全国范围内开展...",
    "news1": "2023-07-28 10:14:27作者：白剑峰来源：人民日报...",
    "questions": "国家卫生健康委员会于2023年7月28日启动了名为...",
    "answers": "「启明行动」是为了防控儿童青少年的近视问题..."
  }
  ```

## 3. 抽样逻辑

由 `backend/scripts/prep_crud_eval.py`（`seed=42`）产出。抽样分布：

| 类别 | 数量 | 来源（CRUD 字段）| 对应主场策略 |
|---|---:|---|---|
| `single_hop`      | 15 | `questanswer_1doc`  | classical / Multi-Query |
| `multi_hop_2`     | 15 | `questanswer_2docs` | Agentic rewrite / Graph local |
| `multi_hop_3`     | 15 | `questanswer_3docs` | Graph RAG hybrid |
| `partial_context` | 10 | `questanswer_3docs`（`gold_ingest_doc_ids` 仅前两篇）| Agentic grading |
| `kb_absent`       |  5 | 合成（虚构实体名如 "张虚构" / "2027 年火星殖民局"）| Agentic full 的 hallucination check |
| **合计** | **60** | | |

实测产出（seed=42，2026-04-22）：

- **60 条 query** 写入 `queries.jsonl`
- **118 篇去重新闻** 写入 `docs/crud-0001.txt ... crud-0118.txt`（去重按 news 文本 sha1）
- 单文件 ~4 KB；总大小 ~472 KB

## 4. 文件路径（在项目中）

```
tests/eval-data/crud-mini/
├── .cache/split_merged.json        # 原始数据（26MB，脚本下载后缓存）
├── docs/crud-NNNN.txt              # 118 篇去重新闻，供 KB 入库
├── queries.jsonl                   # 60 条 query，每行一条
└── metadata.json                   # 抽样 seed / 分布 / 时间戳
```

### `queries.jsonl` schema

每行一条 JSON，字段：

```json
{
  "qid": "Q001",
  "query": "潮宏基珠宝联合哪个设计工作室推出了新品花丝云起系列...",
  "category": "single_hop",
  "gold_answer": "潮宏基珠宝联合 MING DESIGN STUDIO 推出了...",
  "gold_doc_ids": ["crud-0089"],            // Recall@K 用
  "gold_ingest_doc_ids": ["crud-0089"],     // 同 gold；partial_context 则是前 2/3 篇
  "source": "questanswer_1doc",             // CRUD 原字段
  "source_id": "64fa9b32b82641eb8ecc0d17"   // CRUD 原 ID，可追溯
}
```

### `metadata.json`

```json
{
  "seed": 42,
  "total_queries": 60,
  "total_unique_docs": 118,
  "category_distribution": {
    "single_hop": 15, "multi_hop_2": 15, "multi_hop_3": 15,
    "partial_context": 10, "kb_absent": 5
  },
  "source": "https://raw.githubusercontent.com/IAAR-Shanghai/CRUD_RAG/.../split_merged.json",
  "generated_at": "2026-04-22T15:14:38"
}
```

## 5. 再生 / 重采样

```bash
cd backend

# 用 seed=42 重新生成（默认）—— 幂等：cache 存在则跳过下载
venv/bin/python scripts/prep_crud_eval.py

# 换 seed
venv/bin/python scripts/prep_crud_eval.py --seed 123

# 强制重新下载原始数据
venv/bin/python scripts/prep_crud_eval.py --force
```

脚本是纯 stdlib（`urllib` + `hashlib` + `json` + `random`），不依赖 `datasets` / `httpx`。
