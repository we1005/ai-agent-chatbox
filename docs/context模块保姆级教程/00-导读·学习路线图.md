# 00 · 导读：上下文工程保姆级教程

> 这套教程不讲"我们的代码长什么样"，而讲**"为什么会长成这样"**。
>
> 我们会从一个新手都能想到的最朴素方案出发——**把所有聊天记录原样拼给大模型**——然后一步步发现它的问题、想办法、踩坑、再改进，直到走到本项目今天的 Context Engine v2。每一步都回答四个问题：
>
> 1. **上一步的方案，缺点是什么？**（痛点）
> 2. **为了解决它，我们想到了什么方法？灵感从哪来？**（思路 + 业界参考）
> 3. **改进效果如何？**（收益）
> 4. **但它又引入了什么新问题？**（→ 下一章的起点）

---

## 这套教程适合谁

- 想理解"LLM 应用为什么需要上下文工程"的后端 / 算法同学
- 接手本项目、想搞懂 `backend/app/services/{condenser,memory_service,context_router,context_assembler}.py` 为什么这么设计的人
- 对 Mem0 / OpenHands / Zep / Letta 这些"记忆系统"好奇、想知道它们底层在干什么的人

**不需要**你预先懂这些组件。每个组件第一次出现时，我们都会从零讲它的底层原理。

## 一个贯穿全程的比喻：上下文 = 大模型的"工作记忆"

把大模型想象成一个**得了顺行性遗忘症的天才**（电影《记忆碎片》主角）：

- 他智商极高，能解决任何当下问题
- 但他**没有长期记忆**——每次对话结束，脑子就清空
- 你能给他的只有一叠**便利贴**（就是 prompt / context window）
- 便利贴有数量上限（token 上限），贴满了就看不下了

这套教程讲的，本质就是一件事：

> **如何用有限的便利贴，让这个天才表现得像一个有完整记忆的人。**

- 第 1 章：便利贴不够贴了，先只贴最近的（滑动窗口）
- 第 2 章：旧便利贴扔之前，先浓缩成一张（摘要）
- 第 3 章：便利贴得有个可靠的"原始档案库"，不能让别人随便塞（Event Stream）
- 第 4 章：跨天的事情得记到一个"长期笔记本"里，下次翻出来（Durable Memory）
- 第 5 章：每次该贴哪些便利贴、贴不下时先撕哪张，得有规矩（Router + Assembler）

## 阅读建议

- **顺序读**。每一章都建立在前一章的"痛点"之上，跳读会丢失因果链。
- 每章结尾有一个 **"本章遗留问题"** 小节，它就是下一章的开场。
- 代码引用都标了 `文件:行号`，想深挖可以直接跳进源码。
- 想看"最终态全景"和工程细节（配置开关、测试覆盖、失败兜底），读隔壁的 `内部工程手册`（项目内部工程参考手册 `analysis-for-backend/context-engine.md`）——那是**参考手册**，本教程是**学习路径**。
